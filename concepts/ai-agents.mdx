An **AI agent** is the voice-powered assistant that conducts calls on your behalf in Consio. You configure the agent's opening message, conversational style, and the tools it has access to during a call. Once assigned to a campaign, the agent handles every outbound (and inbound) call autonomously — answering questions, looking up orders, sending follow-up messages, and even transferring to a human when needed. You can create multiple agents and assign a different agent to each campaign to match tone, purpose, or brand.

## Core Configuration

Every AI agent is defined by three foundational fields:

### `greeting_message`

The first thing the agent says when a customer picks up the phone. Your greeting should:

- Identify the brand clearly
- State the purpose of the call in plain language
- Sound natural when spoken aloud — avoid long, complex sentences

**Example:**
```text
Hi, this is Jamie calling from Acme Running Co! I'm reaching out because you recently picked up a pair of our trail runners, and I wanted to make sure you're loving them. Do you have a quick minute?
```

### `guidelines`

A set of instructions the agent follows throughout the entire conversation. Use guidelines to define what the agent should and shouldn't do — handling objections, staying on topic, tone adjustments, or specific compliance language.

**Example:**
```text
- Always confirm the customer's name before discussing their order.
- Never mention competitor brands.
- If the customer asks about a return, collect their order number and let them know a human agent will follow up within 24 hours.
- Always end the call by thanking the customer by name.
```

### `personality`

A natural-language description of how the agent should sound and feel to the customer — its overall tone and communication style.

**Example:**
```text
Warm, upbeat, and conversational. Speaks like a knowledgeable friend, not a salesperson. Uses concise sentences and avoids jargon.
```

## Agent Capabilities

You can enable or disable individual tools that expand what your AI agent can do mid-call. Enable only the tools relevant to your campaign to keep the agent focused.

<CardGroup cols={2}>
  <Card title="Find Order" icon="magnifying-glass">
    **`find_order_enabled`**

    Allows the agent to look up a customer's recent order status in real time. When a customer asks "Where is my order?", the agent retrieves the order details and reads them back without needing to transfer the call.
  </Card>
  <Card title="Send SMS" icon="message">
    **`send_sms_enabled`**

    Allows the agent to send a follow-up SMS to the customer during or after the call. Useful for sharing discount codes, tracking links, or a summary of what was discussed.
  </Card>
  <Card title="Search Product Catalog" icon="tag">
    **`search_product_catalog_enabled`**

    Allows the agent to search your connected product catalog to answer questions about availability, pricing, and product details. Keeps the conversation going without escalating to a human for basic product queries.
  </Card>
  <Card title="Create Draft Order" icon="cart-shopping">
    **`create_draft_order_enabled`**

    Allows the agent to create a draft Shopify order on behalf of the customer, including applying a discount code. Ideal for campaigns where the goal is to close a sale or recover an abandoned cart during the call.
  </Card>
  <Card title="Transfer Call" icon="phone-arrow-right">
    **`transfer_call_enabled`**

    Allows the agent to transfer the live call to a human agent when the customer requests it or the conversation requires human intervention. See the [Call Transfer](#call-transfer) section below for configuration details.
  </Card>
</CardGroup>

## Call Transfer

When `transfer_call_enabled` is turned on, you configure where inbound transfers are routed. Consio supports two transfer target types:

**Internal phone number** (`InternalPhoneNumberToTransferTo`) — Transfer to a phone number already registered in your Consio workspace. You reference it by its `phone_number_id`. This is the recommended approach because it keeps routing centralized in your workspace settings.

```json
{
  "type": "InternalPhoneNumberToTransferTo",
  "phone_number_id": "pn_01abc123"
}
```

**External phone number** (`ExternalPhoneNumberToTransferTo`) — Transfer to any external number. You provide the raw phone number and a human-readable description so your team understands where calls are going.

```json
{
  "type": "ExternalPhoneNumberToTransferTo",
  "number": "+18005551234",
  "description": "East Coast Support Team"
}
```

<Tip>
  **Writing effective guidelines and greeting messages**

  - **Keep the greeting under 30 seconds when read aloud.** Long openings cause hang-ups before the conversation starts.
  - **Write guidelines as a numbered or bulleted list**, not flowing prose. The agent processes them better and they're easier for you to maintain.
  - **Be explicit about escalation paths.** If the agent should transfer when a customer mentions a refund or complaint, say so directly in the guidelines: *"If the customer mentions a refund or is upset, immediately transfer the call."*
  - **Test with edge cases.** Think about what a frustrated customer or an unusual question looks like, and add a guideline for it.
  - **Avoid negative instructions alone.** Instead of only saying what the agent shouldn't do, pair it with what it should do instead: *"Don't discuss pricing — offer to send a link via SMS."*
</Tip>

## Agents in API Responses

When you retrieve a call record from the API, the agent that handled the call is included under the `ai_agent` field. This lets you filter and analyze performance by agent across your call history.

```json
{
  "id": "call_01xyz",
  "status": "completed",
  "ai_agent": {
    "id": "agent_01abc",
    "name": "Jamie – Retention Agent",
    "greeting_message": "Hi, this is Jamie from Acme Running Co..."
  }
}
```

A **call record** in Consio is a complete account of a single phone interaction between your AI agent and a customer. Every call — whether dialed by a campaign, triggered manually, or received as an inbound call — generates a record that includes the call status, a full transcript, an AI-generated summary, suggested next steps, and any revenue attributed to the interaction. You can retrieve call records via the REST API to power dashboards, trigger automations, or feed data into your BI tools.

## Call Directions and Trigger Types

Calls in Consio have a **direction** and a **trigger type** that together tell you how the call originated.

| Direction  | Trigger Type        | Description                                                                                       |
| ---------- | ------------------- | ------------------------------------------------------------------------------------------------- |
| `outbound` | `outbound_power`    | The call was automatically dialed by a running campaign (power dialer mode).                      |
| `outbound` | `outbound_click`    | The call was manually triggered by a team member clicking to call a contact from the dashboard.   |
| `inbound`  | `inbound`           | The customer called your Consio phone number and the AI agent answered.                           |

## Call Status Lifecycle

An outbound call moves through a series of statuses from the moment it is scheduled to the moment it concludes. The chart below shows the main path and all possible terminal states.

```text
scheduled → assigned → reserved → dialing → in_progress → completed

Terminal states (can occur at various points before completion):
  excluded       — contact skipped by a campaign rule before dialing
  dnc_excluded   — contact suppressed by Do Not Call list before dialing
  canceled       — call canceled before it connected
  no_agent       — no AI agent available at the scheduled time
  busy           — customer's line was busy
  not_answered   — call rang but customer did not pick up
  failed         — call could not be connected due to a technical error
```

| Status          | Description                                                                                                  |
| --------------- | ------------------------------------------------------------------------------------------------------------ |
| `scheduled`     | The call has been queued and is waiting to be assigned to an agent when the contact window opens.            |
| `assigned`      | The call has been assigned to a specific AI agent and is preparing to dial.                                  |
| `reserved`      | A channel has been reserved; the call is about to go out.                                                    |
| `dialing`       | The phone is actively ringing on the customer's end.                                                         |
| `in_progress`   | The customer answered and the AI agent is in an active conversation.                                         |
| `completed`     | The call ended normally after a live conversation.                                                           |
| `no_agent`      | No AI agent was available to handle the call at the scheduled time.                                          |
| `failed`        | The call could not be connected due to a technical error.                                                    |
| `canceled`      | The call was canceled before it connected — for example, because the campaign was paused.                    |
| `busy`          | The customer's line was busy.                                                                                |
| `excluded`      | The contact was excluded from this call attempt by a campaign rule (e.g. outside contact hours).             |
| `not_answered`  | The call rang but the customer did not pick up (and no voicemail was left, or voicemail was not detected).  |
| `dnc_excluded`  | The contact is on your Do Not Call list and was automatically suppressed.                                    |

## Call Status Filter Groups

The Consio API provides pre-built filter groups that make it easy to query for common subsets of calls without building complex status filters yourself.

**Queued calls:**

| Filter              | Description                                                              |
| ------------------- | ------------------------------------------------------------------------ |
| `all_queued`        | All calls currently in the queue (scheduled, assigned, or reserved).     |
| `in_progress`       | Calls currently being handled by the AI agent.                           |
| `ready_to_call`     | Calls that are scheduled and ready to dial immediately.                  |
| `scheduled_for_later` | Calls scheduled to dial at a future time.                              |
| `off_hours`         | Calls that are queued but waiting for the next contact window to open.   |

**Completed calls:**

| Filter                    | Description                                                                        |
| ------------------------- | ---------------------------------------------------------------------------------- |
| `all_completed`           | Every call that has reached a terminal status.                                     |
| `connected`               | Calls where the customer answered and had a conversation.                          |
| `unanswered`              | Calls where the customer did not pick up (busy, not_answered).                     |
| `failed`                  | Calls that failed due to a technical error.                                        |
| `with_connected_revenue`  | Completed, connected calls that have at least one attributed order.                |
| `with_revenue`            | All completed calls (any terminal status) with at least one attributed order.      |

**Metadata counts:**

| Filter                  | Description                                                    |
| ----------------------- | -------------------------------------------------------------- |
| `total`                 | Total count of all call records.                               |
| `ringing`               | Calls currently in the `dialing` status.                       |
| `in_progress_connected` | Calls currently in an active, live conversation.               |
| `missed`                | Calls that ended without a live conversation.                  |
| `answered`              | Calls where the customer picked up.                            |

## Key Call Record Fields

Each call record returned by the API includes the following fields:

| Field          | Type             | Description                                                                                          |
| -------------- | ---------------- | ---------------------------------------------------------------------------------------------------- |
| `id`           | string           | Unique identifier for the call.                                                                      |
| `status`       | string           | Current or final status of the call (see statuses above).                                            |
| `direction`    | string           | `inbound` or `outbound`.                                                                             |
| `trigger_type` | string           | How the call was initiated: `outbound_click`, `outbound_power`, or `inbound`.                        |
| `customer_id`  | string           | The ID of the contact who was called.                                                                |
| `campaign_id`  | string           | The campaign this call belongs to (if applicable).                                                   |
| `user_id`      | string           | The team member who triggered the call, for `outbound_click` calls.                                  |
| `duration`     | integer          | Call duration in seconds (populated after the call ends).                                            |
| `transcript`   | string           | Full text transcript of the conversation between the agent and the customer.                         |
| `summary`      | array of strings | AI-generated bullet-point summary of the key topics covered in the call.                             |
| `next_steps`   | array of strings | AI-suggested follow-up actions based on the conversation.                                            |
| `revenue`      | number           | Total revenue attributed to this call from linked orders.                                            |
| `recordings`   | array of objects | Call and voicemail recordings associated with this call. Each object has a `type` (`call` or `voicemail`), a URL, and a `file_deleted_at` timestamp. |

<Note>
  Recording files are only available for a **limited time** after the call ends. Each recording object includes a `file_deleted_at` field that tells you exactly when the file will be removed. If you need to retain recordings for compliance or quality assurance purposes, download and archive them before this date.
</Note>

## Accessing Call History via the API

Retrieve your workspace's call history with a `GET` request to the calls endpoint:

```http
GET https://api-martin.consio.dev/public/v1/{workspace_slug}/calls
Authorization: Bearer csk_...
```

You can filter results using query parameters including `status`, `direction`, `campaign_id`, `start_date`, `end_date`, and the pre-built filter groups described above.

For the full list of parameters, response fields, and example responses, see the [Calls API Reference](/api-reference/calls).

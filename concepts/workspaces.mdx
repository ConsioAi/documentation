A **workspace** is the top-level organizational unit in Consio. Everything you do in Consio — launching campaigns, configuring AI agents, managing phone numbers, and reviewing call history — lives inside a workspace. Think of it as your brand's dedicated environment within the platform. When you sign up, Consio creates a default workspace for you. If you manage multiple brands or business units, each one gets its own workspace with completely isolated data.

## Finding Your Workspace Slug

Every workspace is identified by a unique string called its **workspace slug**. You'll need this slug to construct every API request you make.

The easiest way to find your slug is to look at the URL when you're logged into the Consio dashboard:

```text
https://app.consio.dev/my-brand/campaigns
                         ^^^^^^^^
                         This is your workspace_slug
```

Your workspace slug is also visible in your workspace settings under **Settings → General**.

## What Lives in a Workspace

Every resource in Consio belongs to exactly one workspace. Here's what you'll find inside yours:

- **Campaigns** — Outbound calling campaigns targeting a synced customer segment
- **AI Agents** — The voice agents that handle calls on your behalf
- **Phone Numbers** — Inbound and outbound numbers assigned to your account
- **Users** — Team members with roles and permissions
- **Integrations** — Connected platforms like Shopify and Klaviyo
- **Call History** — A complete record of every inbound and outbound call

## User Roles

Workspaces support three user roles. Each role controls what a team member can view and manage within the workspace.

| Role    | Description                                                                                      |
| ------- | ------------------------------------------------------------------------------------------------ |
| `owner` | Full control over the workspace, including billing, workspace deletion, and all settings.        |
| `admin` | Can manage campaigns, AI agents, phone numbers, integrations, and users. Cannot delete the workspace. |
| `rep`   | Read-only access to call history and campaign performance. Useful for sales team members reviewing their calls. |

## API Scoping

Every Consio REST API endpoint is scoped to a specific workspace. The `workspace_slug` appears directly in the URL path, immediately after the API version prefix:

```text
https://api-martin.consio.dev/public/v1/{workspace_slug}/<resource>
```

For example, to list all calls for a workspace with the slug `my-brand`, you would call:

```http
GET https://api-martin.consio.dev/public/v1/my-brand/calls
Authorization: Bearer csk_...
```

This scoping applies to every resource — campaigns, agents, contacts, attributed orders, and more. There is no global API endpoint that returns data across multiple workspaces.

<Note>
  Your API key (`csk_…`) is scoped to a **single workspace**. You cannot use one API key to access data from a different workspace. If you need programmatic access to multiple workspaces, generate a separate API key for each one from that workspace's settings page.
</Note>

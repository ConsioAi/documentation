In this guide you'll authenticate with the Consio API and retrieve a paginated list of calls from your workspace. By the end, you'll have a working request you can adapt for your own integration — no SDKs or special tooling required, just an API key and an HTTP client.

<Steps>

  <Step title="Get your API key">
    All requests to the Consio API must be authenticated with a Bearer token. To generate one:

    1. Log in to the [Consio dashboard](https://app.consio.dev).
    2. Navigate to **Settings → API Keys**.
    3. Click **Generate new key**, give it a descriptive name, and confirm.
    4. Copy the key immediately — it starts with `csk_` and is only shown once.

    Store the key somewhere safe such as a password manager or a secrets vault. You'll pass it in the `Authorization` header of every request.
  </Step>

  <Step title="Note your workspace slug">
    Every API endpoint is scoped to a workspace. Your workspace slug appears in the Consio dashboard URL — it's the short identifier that follows the domain:

    ```text
    https://app.consio.dev/workspaces/my-brand/...
                                       ^^^^^^^^
                                       workspace slug
    ```

    In this example, the slug is `my-brand`. You'll substitute your own slug wherever you see `your_workspace_slug` in the examples below.

    <Note>
      You can also find your workspace slug on the **Settings → General** page of your Consio dashboard.
    </Note>
  </Step>

  <Step title="Make your first request">
    Call the `GET /v1/{workspace_slug}/calls` endpoint to retrieve the first page of call records for your workspace. Replace `your_workspace_slug` and `csk_your_api_key` with your real values.

    <CodeGroup>

    ```bash curl
    curl -X GET \
      'https://api-martin.consio.dev/public/v1/your_workspace_slug/calls?page=1&page_size=20' \
      -H 'Authorization: Bearer csk_your_api_key'
    ```

    ```python Python
    import requests

    workspace_slug = "your_workspace_slug"
    api_key = "csk_your_api_key"

    response = requests.get(
        f"https://api-martin.consio.dev/public/v1/{workspace_slug}/calls",
        headers={"Authorization": f"Bearer {api_key}"},
        params={"page": 1, "page_size": 20},
    )
    data = response.json()
    print(data["pagination"])
    print(data["items"])
    ```

    ```javascript JavaScript
    const workspaceSlug = "your_workspace_slug";
    const apiKey = "csk_your_api_key";

    const response = await fetch(
      `https://api-martin.consio.dev/public/v1/${workspaceSlug}/calls?page=1&page_size=20`,
      {
        headers: { Authorization: `Bearer ${apiKey}` },
      }
    );
    const data = await response.json();
    console.log(data.pagination);
    console.log(data.items);
    ```

    </CodeGroup>
  </Step>

  <Step title="Understand the response">
    A successful response returns HTTP `200` with a JSON body containing two top-level keys: `pagination` and `items`.

    ```json
    {
      "pagination": {
        "page": 1,
        "page_size": 20,
        "total_items": 84,
        "total_pages": 5,
        "next_page": 2,
        "previous_page": null
      },
      "items": [
        {
          "id": "call_01j9z3k...",
          "direction": "outbound",
          "status": "completed",
          "duration_seconds": 142,
          "from": "+14155550100",
          "to": "+14155550199",
          "started_at": "2024-11-01T10:32:00Z",
          "ended_at": "2024-11-01T10:34:22Z",
          "campaign_id": "camp_01j9z...",
          "workspace_slug": "your_workspace_slug"
        }
      ]
    }
    ```

    **Pagination fields**

    | Field | Type | Description |
    |---|---|---|
    | `page` | integer | The current page number. |
    | `page_size` | integer | The number of items returned per page. |
    | `total_items` | integer | Total number of calls matching the query. |
    | `total_pages` | integer | Total number of pages available. |
    | `next_page` | integer \| null | The next page number, or `null` if you're on the last page. |
    | `previous_page` | integer \| null | The previous page number, or `null` if you're on the first page. |

    Iterate through pages by incrementing the `page` query parameter until `next_page` is `null`.
  </Step>

</Steps>

<Note>
  **Rate limits and best practices:** Avoid polling the API more frequently than your use case requires. When syncing large volumes of call data, page through results sequentially rather than making many parallel requests. If you receive a `429 Too Many Requests` response, back off and retry after the interval specified in the `Retry-After` header.
</Note>

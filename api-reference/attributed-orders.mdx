title: "Retrieve Attributed Orders for a Workspace | Consio"
sidebarTitle: "Attributed Orders"
description: "Paginated attributed orders for a workspace. Links Shopify orders to calls, messages, or voicemails. Requires start_date and end_date."
---

Fetch a paginated list of orders that Consio has attributed to a call, message, or voicemail within your workspace. Each record links a Shopify order to the specific interaction that influenced it, giving you a direct view of how your AI-powered outreach drives revenue. Use this endpoint to build reporting dashboards, audit attribution logic, or export data for downstream analytics.

```
GET https://api-martin.consio.dev/public/v1/{workspace_slug}/attributed-orders
```

<Note>
  Unlike the [Calls](/api-reference/calls) endpoint, `start_date` and `end_date` are **required** for this endpoint. Every request must include both parameters to define the reporting window.
</Note>

<Note>
  All requests must include a valid API key in the `Authorization` header. See [Authentication](/api-reference/authentication) for details on generating and managing your API keys.
</Note>

---

## Path Parameters

<ParamField path="workspace_slug" type="string" required>
  The unique identifier for your Consio workspace. You can find this in your workspace settings or in the URL when logged in to the Consio dashboard. See [Workspaces](/concepts/workspaces) for more details.
</ParamField>

---

## Query Parameters

### Date Range (Required)

<ParamField query="start_date" type="datetime" required>
  The start of the reporting window. Only orders attributed to interactions that occurred on or after this timestamp will be returned. Must be a valid ISO 8601 datetime string (e.g., `2024-01-01T00:00:00Z`).
</ParamField>

<ParamField query="end_date" type="datetime" required>
  The end of the reporting window. Only orders attributed to interactions that occurred on or before this timestamp will be returned. Must be a valid ISO 8601 datetime string (e.g., `2024-01-31T23:59:59Z`).
</ParamField>

### Attribution Source

<ParamField query="attribution_source" type="string">
  Filter attributed orders by the type of interaction that influenced them. Accepted values:
  - `call` — orders attributed to a voice call
  - `message` — orders attributed to an SMS or messaging interaction
  - `voicemail` — orders attributed to a voicemail left by an AI agent

  Omit this parameter to return all attributed orders regardless of source. See [Order Attribution](/concepts/order-attribution) for details on how Consio assigns attribution.
</ParamField>

### Pagination & Ordering

<ParamField query="page" type="integer" default="1">
  The page number to return. Must be greater than `0`.
</ParamField>

<ParamField query="page_size" type="integer" default="50">
  The number of records to return per page. Must be greater than `0` and no larger than `100`.
</ParamField>

<ParamField query="order_field" type="string" default="">
  The field name to sort results by. Leave empty to use the default sort order.
</ParamField>

<ParamField query="order_direction" type="string" default="asc">
  The sort direction. Accepted values: `asc` (ascending) or `desc` (descending).
</ParamField>

### Search

<ParamField query="search_term" type="string" default="">
  A free-text search string matched against customer name, order name, and other attribution metadata.
</ParamField>

### Agent Filters

<ParamField query="user_id" type="integer">
  Filter attributed orders to those linked to calls or interactions handled by a specific human agent, identified by their user ID.
</ParamField>

<ParamField query="ai_agent_id" type="integer">
  Filter attributed orders to those linked to interactions handled by a specific AI agent, identified by its agent ID. See [AI Agents](/concepts/ai-agents) for more details.
</ParamField>

## Example Request

<CodeGroup>

```bash cURL
curl -X GET \
  'https://api-martin.consio.dev/public/v1/your_workspace_slug/attributed-orders?start_date=2024-01-01T00:00:00Z&end_date=2024-01-31T23:59:59Z&attribution_source=call&page=1&page_size=50' \
  -H 'Authorization: Bearer csk_your_api_key'
```

```python Python
import requests

workspace_slug = "your_workspace_slug"
api_key = "csk_your_api_key"

response = requests.get(
    f"https://api-martin.consio.dev/public/v1/{workspace_slug}/attributed-orders",
    headers={"Authorization": f"Bearer {api_key}"},
    params={
        "start_date": "2024-01-01T00:00:00Z",
        "end_date": "2024-01-31T23:59:59Z",
        "attribution_source": "call",
        "page": 1,
        "page_size": 50,
    },
)
data = response.json()
```

```javascript JavaScript
const workspaceSlug = "your_workspace_slug";
const apiKey = "csk_your_api_key";

const params = new URLSearchParams({
  start_date: "2024-01-01T00:00:00Z",
  end_date: "2024-01-31T23:59:59Z",
  attribution_source: "call",
  page: 1,
  page_size: 50,
});

const response = await fetch(
  `https://api-martin.consio.dev/public/v1/${workspaceSlug}/attributed-orders?${params}`,
  {
    method: "GET",
    headers: {
      Authorization: `Bearer ${apiKey}`,
    },
  }
);

const data = await response.json();
```

</CodeGroup>

## Response Fields

### Pagination

For full details on navigating multi-page results, see [Pagination](/api-reference/pagination).

### Attribution Object (`items[]`)

<ResponseField name="id" type="integer">
  The unique identifier for this attribution record.
</ResponseField>

<ResponseField name="type" type="string">
  The type of interaction that was attributed to this order. Possible values: `call`, `message`, `voicemail`. See [Order Attribution](/concepts/order-attribution) for a full explanation of how Consio determines attribution.
</ResponseField>

<ResponseField name="call_id" type="integer | null">
  The ID of the call record linked to this attribution. Populated when `type` is `call` or `voicemail`. `null` for message-attributed orders.
</ResponseField>

<ResponseField name="message_id" type="integer | null">
  The ID of the message record linked to this attribution. Populated when `type` is `message`. `null` for call-attributed orders.
</ResponseField>

<ResponseField name="order_total_price" type="number">
  The total value of the attributed order in the workspace's currency (e.g., `149.99`).
</ResponseField>

<ResponseField name="call_in_progress_at" type="datetime">
  The ISO 8601 timestamp at which the attributed interaction (call, message, or voicemail) was in progress.
</ResponseField>

<ResponseField name="order_created_at" type="datetime">
  The ISO 8601 timestamp at which the Shopify order was created.
</ResponseField>

<ResponseField name="cancelled_at" type="datetime | null">
  The ISO 8601 timestamp at which the order was cancelled, if applicable. `null` if the order has not been cancelled.
</ResponseField>

<ResponseField name="financial_status" type="string | null">
  The payment status of the order as reported by Shopify (e.g., `paid`, `pending`, `refunded`, `partially_refunded`).
</ResponseField>

<ResponseField name="customer" type="object">
  The customer who placed the attributed order.

    </ResponseField>

    </ResponseField>
  </Expandable>

  <Expandable title="user">

The Consio API uses **Bearer token authentication**. Every request must include an `Authorization` header containing a valid API key. Keys are prefixed with `csk_` (Consio Secret Key) so you can identify them at a glance in your environment configuration. There are no session cookies, OAuth flows, or short-lived tokens — your `csk_` key is all you need.

## Generate an API Key

1. Log in to the [Consio dashboard](https://app.consio.dev).
2. Click your workspace name in the top navigation, then go to **Settings → API Keys**.
3. Click **Generate new key** and give it a descriptive label (for example, `data-warehouse-sync` or `shopify-integration`).
4. Copy the key as soon as it appears. For security reasons, **Consio does not display the full key again** after you close the dialog.

<Warning>
  Your API key grants full read access to all data in your workspace, including call records, attributed orders, and customer information. Treat it like a password — never share it publicly, never commit it to source control, and rotate it immediately if you suspect it has been compromised.
</Warning>

## Pass the Key in Every Request

Include your API key as a Bearer token in the `Authorization` header of every HTTP request:

```http
Authorization: Bearer csk_your_api_key
```

The following examples show how to add this header in common HTTP clients:

<CodeGroup>

```bash curl
curl -X GET \
  'https://api-martin.consio.dev/public/v1/your_workspace_slug/calls' \
  -H 'Authorization: Bearer csk_your_api_key'
```

```python Python
import requests

workspace_slug = "your_workspace_slug"
api_key = "csk_your_api_key"

headers = {"Authorization": f"Bearer {api_key}"}

response = requests.get(
    f"https://api-martin.consio.dev/public/v1/{workspace_slug}/calls",
    headers=headers,
)
data = response.json()
```

```javascript JavaScript
const workspaceSlug = "your_workspace_slug";
const apiKey = "csk_your_api_key";

const response = await fetch(
  `https://api-martin.consio.dev/public/v1/${workspaceSlug}/calls`,
  {
    headers: {
      Authorization: `Bearer ${apiKey}`,
    },
  }
);
const data = await response.json();
```

</CodeGroup>

## Authentication Error Responses

If your request is missing a key, uses an invalid key, or uses a key that doesn't belong to the workspace in the URL, the API returns an error response. The table below describes the authentication and validation errors you may encounter:

| HTTP Status | Meaning |
|---|---|
| `401 Unauthorized` | The `Authorization` header is missing, malformed, or contains an invalid API key. |
| `403 Forbidden` | The API key is valid but does not belong to the workspace specified in the URL path. |
| `422 Unprocessable Entity` | The API key format is correct but request parameters failed validation. Check the response body for field-level error details. |

When you receive a `401` or `403`, verify that you copied the full key (including the `csk_` prefix) and that the workspace slug in the URL matches the workspace the key was created in.

<Tip>
  Store your API key in an environment variable such as `CONSIO_API_KEY` and read it at runtime. Never hardcode keys directly in source files. This keeps secrets out of version control and makes it easy to rotate keys without changing your code.

  ```bash Shell
  export CONSIO_API_KEY="csk_your_api_key"
  ```

  Then reference it in your application:

  <CodeGroup>

  ```python Python
  import os
  api_key = os.environ["CONSIO_API_KEY"]
  ```

  ```javascript JavaScript
  const apiKey = process.env.CONSIO_API_KEY;
  ```

  </CodeGroup>
</Tip>

## Keep Your API Keys Secure

Follow these practices to minimise the risk of an API key being exposed or misused:

- **Use one key per integration.** Create a separate key for each service or environment (development, staging, production) so you can revoke a single key without disrupting other systems.
- **Never commit keys to source control.** Use `.env` files (excluded via `.gitignore`) or a secrets manager such as AWS Secrets Manager, HashiCorp Vault, or Doppler.
- **Rotate keys regularly.** Treat key rotation as routine maintenance, not just an incident response. Generate a new key, update your configuration, and then delete the old key.
- **Audit key usage.** Review the API Keys page in your dashboard periodically and remove any keys that are no longer in use.
- **Revoke immediately if compromised.** If you believe a key has been exposed, go to **Settings → API Keys** in the Consio dashboard, find the key, and click **Revoke** immediately.
